SOP Before Automation

2026-03-03

I used to start with automation first. But after trying it a few times, I noticed the same pattern: more tools, not much faster execution.

So I changed the order. I now write SOPs before I automate.

This week was simple: break recurring work into units, lock the format with templates, and define a clear DoD for each step.

The practical changes were immediate:

  • Less hesitation when starting work
  • Less quality variance on repeated tasks
  • Less time spent asking “what should I do next?”

The biggest gain showed up in the morning briefing flow. Once the format was fixed, recovery got faster when quality dipped, and adding a one-line day-over-day delta became easy.

I keep one rule: if an SOP can’t produce one concrete action, rewrite it or remove it.
